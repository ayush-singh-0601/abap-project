Problem Statement

In day-to-day procurement operations, users often need to review purchase order details, vendor names, material descriptions, planned delivery dates, and value information quickly. In many cases this information is spread across more than one SAP screen or transaction. As a result, users spend more time navigating than analyzing. This project addresses that issue by building custom ALV reports that present critical purchasing data in business-friendly outputs.

The problem is especially relevant for buyers and procurement coordinators who need to track due items, identify delays, evaluate vendor reliability, and review orders efficiently. Consolidated and searchable reports improve visibility and support faster follow-up actions.

Project Objective

The objective of this project is to create a set of executable ABAP reports centered on ZMM_PO_ANALYTICS_ALV that read standard SAP MM purchasing data and display it through ALV grids with meaningful filters, calculated delivery status, drilldown capability, vendor performance metrics, and delivery tracking.

Main outcome: A suite of medium-complexity, working ABAP programs that demonstrate real development capability and are practical enough for capstone evaluation and verbal explanation during the project test.

Solution Overview

The solution consists of three ABAP reports and three include programs, all designed using CL_SALV_TABLE for ALV output. Each report provides a selection screen for procurement-related filters and retrieves data from standard SAP tables. The codebase follows modular ABAP conventions with separate includes for type definitions, form routines, and local classes.

Functional Features

  • Selection screen filters for company code, purchasing organization, vendor, purchase order number, material, and purchase order date.
  • Integrated view of PO header and item information with vendor and material descriptions.
  • Delivery schedule enrichment using standard schedule-line data from EKET.
  • Status classification as Deleted, Completed, No Schedule, Overdue, Due Soon, or On Time.

Data Flow and Logic

The main report begins by collecting filter values from the selection screen. It reads purchase order data from EKKO and EKPO, vendor details from LFA1, material descriptions from MAKT, and schedule-line delivery dates from EKET. The records are merged inside the ABAP program using Open SQL joins and prepared for ALV display.

Status logic is applied item by item. If an item is marked for deletion, the report displays Deleted. If delivery is completed, it displays Completed. If no schedule line exists, it displays No Schedule. When a valid delivery date is available, the report compares it against the current system date to classify the item as Overdue, Due Soon, or On Time.

The vendor scorecard aggregates item-level data by vendor, counting distinct POs, summing net values, and computing an on-time delivery percentage. The delivery tracker focuses on active items with valid delivery dates, calculating a days-remaining counter and assigning urgency levels (HIGH, MEDIUM, LOW) for quick visual prioritization.

This project directly reflects day-to-day ABAP enhancement and reporting work. It demonstrates selection-screen design, Open SQL joins with multiple tables, internal table processing, ALV configuration, local class-based event handling, and business rule implementation. The addition of a vendor scorecard and delivery tracker shows the ability to build related tools within the same functional domain. At the same time, it avoids unnecessary complexity such as custom database modeling or advanced workflow integration, keeping the scope realistic and manageable.
